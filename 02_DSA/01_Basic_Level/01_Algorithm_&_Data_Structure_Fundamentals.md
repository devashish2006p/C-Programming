# 1. About Data Structure
Data Structure ek aisa systematic tarika hota hai jisme data ko memory me organize, store aur manage kiya jaata hai taaki program us data ko efficiently access, modify, search aur process kar sake.
Matlab simple words me:
jab program ke paas bahut saara data hota hai, to us data ko random ya unordered tarike se memory me rakhne ke bajaye ek structured format me arrange kiya jaata hai, jisse operations jaise insert, delete, search, update aur traversal fast aur efficient ho jaate hain — isi organized format ko data structure kehte hain.


## 1.1 Types of Data Structures 
Broadely there are two types of data structures 

### 1.1.1 Linear Data Structure 
Linear Data Structure wo structure hota hai jisme data elements ek single sequential order (ek line) me arranged hote hain, jahan har element ka maximum ek predecessor (previous) aur ek successor (next) hota hai, isliye data ko start se end tak step-by-step traverse kiya ja sakta hai.

### 1.1.2 Non-Linear Data Structure 
Non-Linear Data Structure wo structure hota hai jisme data elements sequential single line me arranged nahi hote, balki hierarchical ya network form me connected hote hain, jahan ek element multiple elements se connected ho sakta hai, isliye data ko simple start-to-end order me traverse karna possible nahi hota.


## 1.2 Types of Linear Data Structure
### 1.2.1 Array 
Array ek linear data structure hota hai jisme same data type ke elements ko contiguous (continuous) memory locations me sequential order me store kiya jaata hai, aur har element ko index ke through directly access kiya ja sakta hai.

### 1.2.2 Linked List
Linked List ek linear data structure hota hai jisme data elements (nodes) memory me contiguous nahi hote, balki har node apne data ke saath next node ka address store karta hai, jisse nodes pointers ke through ek chain ki tarah connected rehte hain.

### 1.2.3 Stack 
Stack ek linear data structure hai jisme data elements sequential order me arranged hote hain aur jisme insertion (push) aur deletion (pop) sirf ek hi end se hota hai jise TOP kaha jaata hai, aur ye structure strictly LIFO (Last In First Out) principle follow karta hai, matlab jo element sabse last me stack me insert hota hai wahi sabse pehle remove hota hai.

### 1.2.4 Queue 
Queue ek linear data structure hai jisme data elements sequentially arranged hote hain aur insertion (enqueue) ek end se hota hai jise rear kehte hain, aur deletion (dequeue) doosre end se hota hai jise front kehte hain, aur ye strictly FIFO (First In First Out) principle follow karta hai, matlab jo element sabse pehle queue me aayega wahi sabse pehle remove hoga.


## 1.3 Types of Non Linear Data Structure 
### 1.3.1 Tree
Tree ek non-linear data structure hai jisme elements (nodes) hierarchical order me arranged hote hain, jisme ek special node hota hai jise root kehte hain, aur har node zero ya more child nodes se connected ho sakta hai, jisse parent-child relationship banta hai, aur koi cycles nahi hoti.

### 1.3.2 Graph 
Graph ek non-linear data structure hai jisme elements ko nodes (ya vertices) kehte hain aur in nodes ko edges ke through connect kiya jaata hai, jisse multiple relationships represent ki ja sakti hain, aur graph me cycles bhi ho sakti hain, aur ek node ke multiple connections (neighbors) ho sakte hain.

# 2. About Algorithm
Algorithm ek step-by-step procedure hai jo kisi specific problem ko solve karne ke liye clearly defined instructions ka sequence follow karta hai aur finite steps me output produce karta hai.

## 2.1 Algorithm design Basics
    1. Problem Understanding: Sabse pehle problem ko dhyan se samjho aur clearly define karo.
    2. Input/Output Definition: Decide karo algorithm ka input kya hoga aur expected output kya generate hoga.
    3. High-Level Approach: Problem ko chhote sub-problems me divide karo aur step-by-step solution ka plan banao.
    4. Select Strategy/Paradigm: Problem ke liye suitable design paradigm choose karo jaise divide and conquer, greedy, dynamic programming, ya backtracking.
    5. Detailed Steps: High-level plan ko concrete aur executable steps me convert karo, jo deterministic aur finite ho.
    6. Testing with Examples: Algorithm ko small input examples pe test karo aur correctness verify karo. 
    7. Optimization: Time aur space complexity evaluate karo aur algorithm ko efficient aur practical banao.

## 2.2 Pseudocode Writing 
Pseudocode writing ek language-independent way hai algorithm ko clearly aur step-by-step represent karne ka, jisse program likhne se pehle logic samajh aata hai.

Flowchart ek diagrammatic representation hota hai jisme kisi process, algorithm, ya program ke steps ko symbols aur arrows ke through visually dikhaya jaata hai, taki logic aur flow easily samajh me aaye.
